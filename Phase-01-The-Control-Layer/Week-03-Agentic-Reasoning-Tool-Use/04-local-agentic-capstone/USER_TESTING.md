# User Testing Documentation

This repo now separates user testing docs into two files:

- `USER_TESTING_PARTICIPANT.md`:
  - Share this with external/internal test participants.
  - Contains only participant-safe instructions and tasks.
- `USER_TESTING_TEAM_PLAYBOOK.md`:
  - Internal facilitator/team operations doc.
  - Contains setup, metrics, issue severity, and exit criteria.

How to use:

1. Share only `USER_TESTING_PARTICIPANT.md` with testers.
2. Keep `USER_TESTING_TEAM_PLAYBOOK.md` internal.
3. Record findings using the templates in the team playbook.
