# User Testing Documentation

This repo uses three user-testing docs:

- `USER_TESTING_PARTICIPANT.md`:
  - Share this with external or internal test participants.
  - Contains the current task script for account creation, conversation switching, rename/delete flows, and logout recovery.
- `USER_TESTING_TEAM_PLAYBOOK.md`:
  - Internal facilitator/team operations doc.
  - Contains setup, metrics, issue severity, exit criteria, and admin-only debug drawer checks.

How to use:

1. Share only `USER_TESTING_PARTICIPANT.md` with testers.
2. Keep `USER_TESTING_TEAM_PLAYBOOK.md` internal.
3. Use this file as the entry point when deciding which testing doc to open.
4. Record findings using the templates in the team playbook.
