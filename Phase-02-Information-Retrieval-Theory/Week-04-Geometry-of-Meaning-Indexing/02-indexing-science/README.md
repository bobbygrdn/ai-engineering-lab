# **Part 2 Indexing Science — Pinecone HNSW Experiments**

## **Terms**

- Vector search: retrieving items by geometric similarity instead of exact keyword matches.
- Embedding: a numeric vector that represents text, images, or other data in semantic space.
- ANN (Approximate Nearest Neighbor): a fast search method that finds likely nearest vectors without checking every vector.
- HNSW (Hierarchical Navigable Small Worlds): a graph-based ANN index that accelerates search by jumping through layered neighborhoods.
- Recall: how often the system returns the truly correct nearest neighbors.
- Latency: how long the query takes.
- Neighborhood / graph traversal: the local candidate-walking process HNSW uses instead of brute-force scanning.
- Free index service: Pinecone or Weaviate, which can host vector indexes and let you experiment with ANN behavior.
- Approximate: not exact by design; the index intentionally trades some correctness for speed.

## **Key Concepts**

- Brute force is exact but slow: checking 1M+ vectors one by one is reliable but expensive.
- ANN is a controlled gamble: you search a subset of candidates, betting that the nearest neighbors are nearby in the index structure.
- HNSW is a shortcut graph: it organizes vectors so search can move from coarse regions to finer local regions quickly.
- Hierarchy reduces search cost: upper layers help you jump near the target area; lower layers refine the answer.
- Recall vs latency is the core trade-off: higher recall usually means more work, which increases latency.
- Tuning changes behavior: parameters like search breadth affect how many candidates are explored, which changes speed and accuracy.
- Engineering judgment matters: the “best” index is not the one with perfect accuracy; it is the one that meets product latency and quality targets.

## **Implementation Overview**

- Language & stack: Python (see `requirements.txt` for dependencies such as `pinecone`, `hnswlib`, `faiss-cpu`, `scikit-learn`, `pandas`, `matplotlib`).
- Primary purpose: sweep HNSW-style index parameters, measure latency and recall, then aggregate and visualize results.
- Primary capabilities discovered in the code:
  - Run an experiment sweep against a Pinecone index and record per-query and per-trial summaries experiment_pinecone.py.
  - Compute ground-truth nearest neighbors via brute-force cosine similarity for recall measurement.
  - Aggregate per-query and per-trial metrics, produce CSV summaries and multiple diagnostic plots analyze_results.py.
  - Save outputs: `pinecone_per_query.csv`, `pinecone_results.csv`, and image plots under `analysis_output/`.

## **How It Works**

1. Prepare: configure environment variables in `.env` (set `PINECONE_API_KEY` and `INDEX_NAME`).
2. Synthesize dataset: `experiment_pinecone.py` samples synthetic vectors, normalizes them, and selects query vectors from the dataset.
3. Ground truth: the script computes brute-force cosine similarity to obtain exact top-K ground-truth ids for recall evaluation.
4. Upsert: vectors are upserted into the Pinecone index (either once or per-combo if `CREATE_INDEX_PER_COMBO` is enabled).
5. Sweep: iterate over a grid of HNSW-like parameters (`M`, `ef_construction`, `ef_search`) and perform warm-up queries followed by timed trials.
6. Record per-query data: for each query, the script logs timestamp, latency (ms), returned matches, and computed `recall_at_k` into `pinecone_per_query.csv`.
7. Record trial-level summaries: per-trial aggregates (avg latency, p95/p99, mean recall) are appended to `pinecone_results.csv`.
8. Analyze: run `analyze_results.py` to merge CSVs, compute pooled latency percentiles, per-query recall summaries, and generate plots comparing recall vs latency.

## **Example Usage**

- Run an experiment (creates `pinecone_per_query.csv` and `pinecone_results.csv`):

```bash
python experiment_pinecone.py
```

- Analyze results and produce plots (writes `analysis_output/aggregated_results.csv` and PNGs):

```bash
python analyze_results.py
```

- Files of interest:
  - `experiment_pinecone.py`
  - `analyze_results.py`
  - `pinecone_per_query.csv`, `pinecone_results.csv`, and `analysis_output/` (generated by scripts)
